Customization
Customization can be done in three ways:
  • HTML template using class attribute
  • Component instance using setConfig() method
  • Dependency Injection using provideDropdownConfig() factory
Customizations precedence

Customizations precedence is defined as follows
Read it this way: left customizations overwrites right customizations.

setConfig() => class attribute => provideDropdownConfig()
  1. Use provideDropdownConfig() factory, to apply customizations at host component or module level.
  2. Use class attribute, to customize a single instance at host component view level.
  3. Use setConfig() method, to customize a single instance at host component class level. (Highest precedence)
